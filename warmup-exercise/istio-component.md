## **Kiali**
- **역할**: 서비스 메시 관리 및 시각화 대시보드
- **주요 기능**:
  - 서비스 간 통신 토폴로지를 그래프로 시각화
  - 트래픽 플로우, 응답 시간, 에러율 등 실시간 메트릭 표시
  - Istio 설정 검증 및 문제점 진단
  - 서비스 메시 전체 상태를 한눈에 모니터링

## **Jaeger**
- **역할**: 분산 추적(Distributed Tracing) 시스템
- **주요 기능**:
  - 마이크로서비스 간 요청 흐름을 추적
  - 각 서비스별 처리 시간과 지연 구간 분석
  - 성능 병목점 식별
  - 복잡한 서비스 호출 체인의 디버깅 지원

## **Zipkin**
- **역할**: Jaeger와 유사한 분산 추적 도구
- **주요 기능**:
  - Jaeger의 대안으로 사용 가능
  - 서비스 간 요청 추적 및 시각화
  - 트레이스 데이터 수집 및 분석
  - 일반적으로 Jaeger와 Zipkin 중 하나를 선택해서 사용

## **실무 활용 팁**
- **Kiali**: 서비스 메시 전반적인 상태를 모니터링할 때 가장 먼저 보는 대시보드
- **Jaeger/Zipkin**: 특정 API 호출이 느리거나 에러가 발생할 때 원인 분석용
- **함께 사용**: 세 도구를 연계하여 종합적인 관찰성(Observability) 확보

처음 사용하실 때는 Kiali로 전체적인 서비스 구조를 파악하신 후, 문제가 생겼을 때 Jaeger로 상세 분석하는 workflow를 추천합니다.
